# Flow การทำงานของระบบ Smart Moto Service Center

ระบบ Smart Moto Service Center ถูกออกแบบมาให้ครอบคลุมการจัดการศูนย์บริการรถจักรยานยนต์ตั้งแต่ลูกค้านำรถเข้ามาซ่อม ไปจนถึงการชำระเงินและรับรถกลับ โดยแบ่งการทำงานตามบทบาท (Role-based) ดังนี้:

---

## 1. ฝ่ายรับรถ (Reception / Service Advisor)
**หน้าที่หลัก:** ต้อนรับลูกค้า บันทึกข้อมูลเบื้องต้น และรับรถเข้าศูนย์
- **จัดการข้อมูลลูกค้าและรถ:** สร้างประวัติลูกค้าใหม่และเพิ่มรถจักรยานยนต์เข้าสู่ระบบ
- **การนัดหมาย (Appointments):** บันทึกการนัดหมายล่วงหน้าของลูกค้า (สามารถแปลงการนัดหมายเป็นใบแจ้งซ่อมได้เมื่อลูกค้ามาถึง)
- **สร้างใบแจ้งซ่อม (Job Order):** เมื่อลูกค้ามาถึงศูนย์ พนักงานรับรถจะประเมินอาการเบื้องต้นและสร้างใบแจ้งซ่อม (Job Order)
  - *สถานะงานเริ่มต้น:* `PENDING` (รอรับงาน)

---

## 2. หัวหน้าช่าง (Foreman)
**หน้าที่หลัก:** บริหารจัดการคิวงานซ่อม ควบคุมคุณภาพ และประเมินการเบิกอะไหล่
- **ตรวจสอบคิวงาน (Job Queue):** ดูรายการรถที่รอรับบริการทั้งหมดผ่าน Dashboard
- **มอบหมายงาน (Assign Job):** เลือกงานที่สถานะ `PENDING` และมอบหมายให้ช่างซ่อม (Technician) ที่ว่าง
- **จัดการ Outsource:** หากเป็นงานเฉพาะทางที่ทำเองไม่ได้ สามารถส่งงานออกไปให้ซ่อมภายนอกผ่านเมนู Outsource ได้
- **ตรวจสอบงาน (QC):** ตรวจสอบงานซ่อมที่เสร็จแล้ว (หากเปิดฟังก์ชันนี้ สถานะจะผ่าน `QC_PENDING`)

---

## 3. ช่างซ่อม (Technician / Mechanic)
**หน้าที่หลัก:** ดำเนินการซ่อมบำรุงตามใบแจ้งซ่อม
- **เริ่มงานซ่อม:** ช่างกด "เริ่มงาน" สถานะงานจะเปลี่ยนจาก `PENDING` เป็น `IN_PROGRESS`
- **บันทึกการตรวจสอบ (Checklist):** บันทึกสภาพรถก่อน-หลังการซ่อม และบันทึกเวลาทำงาน (Labor Time) พร้อมค่าแรง
- **การขอเบิกอะไหล่ (Part Requisition):**
  - หากต้องใช้อะไหล่ ช่างจะเข้าไปที่หน้ารายละเอียดงานและทำการ **ขอเบิกอะไหล่** 
  - *สถานะงานเมื่อขอเบิก:* `WAITING_PARTS` (รออะไหล่)
- **เสร็จสิ้นงานซ่อม:** หลังจากได้รับอะไหล่และประกอบเสร็จเรียบร้อย ช่างจะกดปุ่ม **"เสร็จสิ้นงาน"**
  - *สถานะงานเมื่อซ่อมเสร็จ:* `COMPLETED` (รอชำระเงิน/รับรถ)

---

## 4. คลังอะไหล่ (Inventory / Stock Keeper)
**หน้าที่หลัก:** จัดการสต๊อกอะไหล่ รับของ และจ่ายของตามใบเบิก
- **จัดการใบเบิกอะไหล่ (Requisitions):** เมื่อช่างส่งคำขอเบิก ฝ่ายคลังจะทำการตรวจสอบและกด "อนุมัติและจ่ายของ" (Approve & Issue) ซึ่งระบบจะไปตัดสต๊อกอะไหล่โดยอัตโนมัติ
- **สั่งซื้ออะไหล่ (Purchase Orders - PO):** ตรวจสอบอะไหล่ที่ใกล้หมดหรือถึงจุดสั่งซื้อ (Reorder Point) และสร้างใบสั่งซื้อ (PO) ส่งไปยังซัพพลายเออร์
- **รับสินค้าเข้าคลัง (Goods Receipt):** เมื่ออะไหล่มาส่ง ฝ่ายคลังจะสร้างใบรับสินค้าเพื่อนำอะไหล่เข้าสต๊อก (เพิ่ม Stock Quantity)
- **จัดการใบเสนอราคาอะไหล่:** เพิ่มแก้ไขราคาและรายละเอียดของอะไหล่ (Part Master)

---

## 5. บัญชีและการเงิน (Accountant / Cashier)
**หน้าที่หลัก:** รับชำระเงิน ออกใบเสร็จ และดูสรุปรายงาน
- **จัดการงานที่ซ่อมเสร็จ:** ตรวจสอบงานที่สถานะ `COMPLETED` (รอชำระเงิน)
- **ออกใบแจ้งหนี้/รับชำระเงิน (Payments):** สรุปค่าแรงซ่อม + ค่าอะไหล่ทั้งหมด นำมาออกบิลและรับชำระเงินจากลูกค้า
  - *สถานะงานเมื่อชำระเงินเรียบร้อย:* `PAID` (ชำระแล้ว - พร้อมรับรถกลับ)
- **ดูรายงานแบบเรียลไทม์:** พิมพ์ใบเสนอราคา (Quotation) หรือดูสรุปรายได้ประจำวัน/เดือน ผ่าน Dashboard บัญชี

---

## 6. เจ้าของร้าน / ผู้จัดการ (Owner / Manager)
**หน้าที่หลัก:** ดูภาพรวมการทำงานทั้งหมด
- **Dashboard:** ดูรายได้ สรุปค่าใช้จ่าย(จากการสั่งซื้ออะไหล่) ยอดขาย และประสิทธิภาพของช่าง
- **จัดการพนักงาน (Employees):** เพิ่ม ลด หรือแก้ไขตำแหน่งของพนักงาน
- **รายงานสรุป (Reports):** ตรวจสอบรายงานประสิทธิภาพและสต๊อกอะไหล่แบบรวมศูนย์

---

## สรุป Flow สถานะงานซ่อม (Job Status Pipeline)
1. **PENDING** - สร้างบิลรับรถเข้า รอจ่ายงานให้ช่าง
2. **IN_PROGRESS** - ช่างกดเริ่มงานและกำลังดำเนินการซ่อม
3. **WAITING_PARTS** - ช่างกดขอเบิกอะไหล่ รอฝ่ายคลังจ่ายของ (เมื่อได้ของแล้วช่างก็กลับไปทำต่อจน IN_PROGRESS)
4. **COMPLETED** - ช่างซ่อมเสร็จเรียบร้อย รอให้ลูกค้ามารับและชำระเงิน
5. **PAID** - รับเงินเรียบร้อย ถือเป็นการสิ้นสุดกระบวนการ

## ข้อมูลเพิ่มเติมสำหรับผู้พัฒนา (Seed Data)
เพื่อประหยัดเวลาในการกรอกข้อมูลทดสอบ ระบบได้เตรียมชุดข้อมูลสาธิต (Full Seed) ที่จำลองเหตุการณ์จริง ครอบคลุมผู้ใช้งาน ลูกค้า อะไหล่ และตัวอย่างงานในทุกๆ สถานะ 
- **คำสั่งรันข้อมูลสาธิต:**
  ```bash
  cd backend
  npm run prisma:seed-full
  ```
- ข้อมูลผู้ใช้งานที่สามารถ Login ได้คือ (รหัสผ่านคือ `password123`):
  - `admin`, `owner1`, `sa1`, `foreman1`, `tech1`-`tech5`, `stock1`, `cashier1`
